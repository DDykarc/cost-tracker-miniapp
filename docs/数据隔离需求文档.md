# 数据隔离方案 — 需求确认

## 当前问题

朋友打开你的小程序后，能看到你之前填的所有数据。

## 原因

微信云数据库**默认权限**是"所有用户可读"——任何人打开小程序都能读到集合里的全部记录，所以朋友能看到你的数据。

## 解决方案

**不需要做登录系统。** 云数据库会自动给每条记录打上 `_openid`（微信用户的唯一标识），我们只需要做两件事：

1. **改权限**：把三个集合的权限从"所有用户可读"改成"仅创建者可读写"
2. **改查询**：代码查询时加上 `where({ _openid: 当前用户 })` 过滤

改完后效果：
- 每个用户**自动隔离**，只能看到自己创建的数据
- **不需要登录**，打开小程序就自动识别身份（微信底层自动处理）
- 你和朋友各自有独立的数据，互不影响

## 需要改的文件和内容

### 1. 数据库集合权限（在云开发控制台操作）

| 集合名 | 当前权限 | 改为 |
|--------|---------|------|
| `items` | 所有用户可读 | 仅创建者可读写 |
| `health_records` | 所有用户可读 | 仅创建者可读写 |
| `version_info` | 所有用户可读 | 仅创建者可读写（或保持"所有用户可读"，版本信息所有人可见） |

### 2. 代码改动

#### `utils/db.js`（回本清单数据服务）
- `getAllItems()`：查询时加 `_openid` 过滤，确保只读自己的数据

#### `utils/healthDb.js`（健康记录数据服务）
- `getRecords()`：查询时加 `_openid` 过滤
- `getRecord()`：查询时加 `_openid` 校验（防止通过 ID 读到别人的数据）

#### `utils/version.js`（版本信息）
- `version_info` 集合可以保持"所有用户可读"，因为版本号和更新日志是公开信息，所有用户都应该能看到

## 不需要做的事

- 不需要登录页面
- 不需要用户注册
- 不需要微信授权弹窗
- 不需要存储用户 token

## 验证方式

1. 你用自己的微信录入几条数据
2. 用朋友的微信打开小程序，应该看到空的列表（自己的数据自己看不到）
3. 朋友录入数据后，你再打开也看不到朋友的数据

## 确认问题

1. 你是否同意"不需要登录系统，只做数据隔离"这个方案？
2. `version_info`（版本信息）是否所有人都能看到，还是也只让创建者看到？
3. 是否需要在设置页面显示当前登录的微信号昵称/头像（仅展示，不做数据存储）？
